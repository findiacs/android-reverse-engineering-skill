
## 2024-05-16 - [Optimize basename in bash loops]
**Learning:** Calling the external command `basename` inside loops in shell scripts creates significant subshell process-forking overhead.
**Action:** Use native bash parameter expansion (e.g., `${var##*/}` for basename, and `${var%.ext}` to remove extensions) to achieve the same result orders of magnitude faster.

## 2024-05-23 - [Optimize multiple greps in bash scripts]
**Learning:** Running multiple separate `grep` commands sequentially over a large directory generates massive I/O overhead as the filesystem traverses the same files repeatedly. Standard `grep` combined caching fails when the regex inadvertently matches the `filename:line:` format generated by `-rn`, creating false positives.
**Action:** Use a two-pass `grep` execution model. Pass 1 constructs a combined regex and runs a single `grep` across the codebase into a secure `mktemp` file. Pass 2 executes specific queries against this much smaller cache file. To prevent false positives from the cached format, bind Pass 2 searches to the source content only by using the regex `:[0-9]+:.*($pattern)`.
